# Audit Log: 2026-06-09

Foundational audit pass against The Pragmatic Programmer principles
(orthogonality, DRY, no broken windows, easy to change), plus a11y,
perf, security. PR: `audit/2026-06-09` → main.

## Shipped in this PR

**Orthogonality (`/coupling`):**
- Verdict: codebase already exemplary. Every component is a leaf module
  (zero component-to-component imports). Library layer reduces to three
  canonical singletons (`SITE_URL`, `buildRouteMetadata`, blog helpers)
  all documented in CLAUDE.md. No cycles, no cross-layer violations,
  no wildcards. Zero findings.

**DRY (`/drift` + `/simplify`):**
- Extracted 7 reusable primitives from `src/app/work/page.tsx` (1192 →
  877 lines, a 27% reduction):
  `<ProjectCard>`, `<ProjectHeader>`, `<ProjectSection>`, `<ProjectText>`,
  `<ProjectBullets>`, `<Checkmark>`, `<TechStackTags>`, `<ExternalLink>`,
  `<MediaButton>`.
- Extracted `BG_BASE`, `SURFACE`, `ACCENT_TEAL` hex constants to
  `src/lib/theme.ts` (previously `#0B0C0E` lived in both `layout.tsx`
  and `globals.css`).
- Documented all primitives in CLAUDE.md "Before Writing" section.

**Broken windows (`/zombie`):**
- Zero dead code, zero dead deps, zero stale imports, zero
  commented-out blocks, zero TODOs / FIXMEs / HACKs, zero empty
  catches. Clean repo.

**Easy to change (efficiency wins):**
- `getPostBySlug` wrapped with React `cache()`. Previously
  `generateMetadata` + the page component both re-read the same MDX
  file from disk; now deduped per request.
- `sitemap.ts` rewritten to use `getAllPosts()` instead of a per-slug
  `getPostBySlug()` loop.
- `VideoBackground` preload changed from `auto` to `metadata`.
  Background loop is ~15MB; previously preloaded on every page.

**Accessibility (`/a11y`):**
- 9 media trigger `<div onClick>` wrappers replaced with the new
  keyboardable `<MediaButton>`. Previously a hard block for
  keyboard-only and switch users.
- Heading hierarchy fix: ProjectHeader rendered h3/h4 beneath the page
  h1, skipping h2. Now h2/h3 (visual styling preserved).
- Desktop Sidebar wrapper changed from `<aside>` to `<nav
  aria-label="Primary">`. Mobile drawer kept as
  `<aside role="dialog">` (correct for a modal nav).
- Decorative SVGs marked `aria-hidden="true"`.
- Media modal now closes on Escape key + locks body scroll while open.

**Security (`/security-review`):**
- `<ExternalLink>` normalizes `target="_blank" rel="noopener noreferrer"`.
  Caught and fixed two latent misses (Nortal + ConnectWise "Visit
  Website" links).
- Audited the 4 `dangerouslySetInnerHTML` call sites; all controlled
  JSON.stringify of typed objects (JSON-LD schema). Safe.

## Deferred (not in this PR)

**`postcss <8.5.10` transitive vulnerability** (`npm audit`):
- 2 moderate severity vulns, both via `next`'s bundled postcss.
- Cannot fix without `npm audit fix --force` which downgrades Next.js
  to 9.3.3 (we're on 16.2.7).
- Fix lives with Next.js; track their release notes.

**OG images are 2MB each** (18 images × ~2MB = 31MB total):
- Loaded by social crawlers and link-preview services only, not page
  loads. Acceptable but compression to ~300KB each would speed up
  social share previews. Content-side task, not code.

**2 blog posts missing custom OG images:**
- `applying-to-the-claude-partner-network`
- `hold-em-or-fold-em`
- Both fall back to `/og-image.png`. Not a bug, just sub-optimal social
  share CTR.

**`StructuredData.tsx` `dateModified` literal:**
- Currently `"2026-03-27"`, hand-typed. Auto-derive from latest git
  commit or build timestamp.

**`#09090b` hex in inline SVG** (`ExitIntentPrompt.tsx`):
- Could be `currentColor` or a theme reference. Cosmetic.

**`surfrrosa.com` literal in 2 sites:**
- Could centralize into an `EXTERNAL_LINKS` constant. Low value at N=2;
  revisit if a third use site appears.
